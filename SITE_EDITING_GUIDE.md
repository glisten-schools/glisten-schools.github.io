# Glisten Schools Website — Easy Editing Guide

This version was reorganised so the most repeated and frequently updated content can be changed from one central file.

## 1. The main file to edit

Open:

`js/site-content.js`

From this one file you can update:

- School names and class levels
- Logos and home-page links
- Phone numbers, WhatsApp links, and emails
- Location and Google Maps link
- Navigation menu items and dropdowns
- Footer contact information
- School highlights and events
- Event dates, descriptions, captions, and image paths

Changes to navigation and footer information automatically appear on every page.

## 2. Adding a new school event

1. Copy the event photos into:

   `assets/images/events/`

2. Open `js/site-content.js`.
3. Copy the existing event object inside the `events` list.
4. Change:
   - `id`
   - `category`
   - `title`
   - `displayDate`
   - `location`
   - `summary`
   - `description`
   - image paths, alternative text, and captions

A new event automatically appears in:

- The homepage highlight section
- The Highlights page
- The Gallery under “School Events”

## 3. Updating the navigation or footer

Edit the `navigation`, `contacts`, `schools`, or `location` sections inside:

`js/site-content.js`

Do not edit the same navbar or footer separately on every HTML page. They are generated by:

`js/components.js`

## 4. Editing page content

Open the relevant page:

- `index.html` — Homepage
- `about.html` — About Us
- `academics.html` — Academics
- `admissions.html` — Admissions
- `contact.html` — Contact Us
- `gallery.html` — Gallery
- `highlights.html` — School highlights and events
- `glisten-preparatory.html` — Glisten Preparatory
- `results-standard-four.html` — Standard Four results
- `results-standard-seven.html` — Standard Seven results

Text inside the main page sections can be edited directly.

## 5. Design and hover effects

The original layout styles are stored in:

`css/style.css`

The newest premium navbar, hover effects, loading indicators, event layouts, and animations are stored in:

`css/premium.css`

The colour variables are at the top of `css/premium.css`. Changing those variables updates the premium colour system across the website.

## 6. Adding normal gallery photographs

1. Add the photograph to `assets/images/gallery/`.
2. Open `gallery.html`.
3. Copy one existing `gallery-item`.
4. Change the image path, alternative text, caption, and label.

For an important school event, use the central event method in section 2 instead.

## 7. Important image practice

- Use clear JPG or WebP images for photographs.
- Keep most website images below about 500 KB where possible.
- Use meaningful file names such as `sports-day-2026-01.jpg`.
- Always add useful `alt` text for accessibility.

## 8. Previewing the website

You can open `index.html` directly in a browser. For the most reliable preview, start a small local server from the website folder:

`python -m http.server 8000`

Then open:

`http://localhost:8000`

## Premium V6: the easiest update workflow

The site now uses three clear layers:

1. **`js/site-content.js` — content you update often**
   - School names, stages, logos, phone numbers, email addresses and WhatsApp links
   - Navigation items
   - School highlights/events and all event images

2. **`js/components.js` — shared layouts**
   - Navbar and dropdowns
   - Footer
   - Homepage event highlight
   - Highlights page event stories
   - Event photographs in the Gallery
   - Contact page school cards

3. **`css/final-premium-v6.css` — final visual style**
   - Coloured navigation strip
   - Hover effects
   - Contact page design
   - Contrast fixes
   - Lightbox controls and loading states

### Add another event or event image
Open `js/site-content.js`, find `events`, copy an existing event or image object, and change the title, date, description, image path, alt text and caption. The homepage, Highlights page and Gallery update automatically.

### Change a phone number or email once
Change it in `js/site-content.js`. The shared navbar/footer and the Contact page cards update automatically.

### Add a new photograph file
Place it inside the appropriate folder under `assets/images/`, then reference that path from `js/site-content.js` or the relevant page.

### Safe editing rule
Keep page-specific wording in its HTML page, repeated school/contact/event information in `js/site-content.js`, shared structure in `js/components.js`, and future design overrides in `css/final-premium-v6.css`.

## Premium Performance V7

### Adding photographs without making the website heavy

1. Copy new JPG or PNG photographs into the correct folder under `assets/images/`.
2. Double-click `OPTIMIZE_NEW_IMAGES.bat` on Windows.
3. The tool creates:
   - a clear full image in WebP format
   - a small card/gallery copy under `assets/images/thumbs/`
4. Use the generated `.webp` path in `js/site-content.js` or the relevant HTML page.

For a new event, continue adding only the full WebP path to `js/site-content.js`; the shared components automatically use the matching thumbnail in cards and the full image in the viewer.

### Image viewer behaviour

- Links carrying `data-lightbox` automatically receive Back, Previous, Next, Close and counter controls.
- Normal content photographs are also enhanced automatically when they are not part of a navigation link.
- Do not remove `js/main.js`, because it controls the viewer, gallery filters, sliders and loading behaviour.
